# Inverse Head & Shoulders（逆三尊）

## 概要

ヘッド・アンド・ショルダーズの逆。**左肩 → 頭 → 右肩** で中央の谷が一番深い。

```
 ───＊              ＊───   ← ネックライン
    / \   肩    / \
   /   \  /＊\  /   \
  /     \/    \/     \
 ＊  左肩 ＊     ＊ 右肩 ＊
          \     /
           \   /
            ＊ 頭（最深）
```

## 意味合い

- いったん大きく売られたがそこを底に戻してくる
- 反転の定番パターン

## 見るポイント

- **右肩が頭ほど深くならない**
- 肩の間の高値を結んだ **ネックラインを上抜け** で買い優勢

## 検出アルゴリズム

1. Pivot 列から `low, high, low, high, low` の 5 ピボットを抽出
   - `L_shoulder.low > head.low`（左肩は頭より浅い）
   - `R_shoulder.low > head.low`（右肩は頭より浅い）
   - `|L_shoulder.low - R_shoulder.low| / head.low <= shoulderBalance`（左右のバランス）
2. ネックライン: 2 つの `high` を結ぶ線分（右肩時点の y を使う簡易実装で可）
3. 確定: 右肩以降の `close` がネックラインを上抜け

## 閾値

| 名称                | 既定  |
|---------------------|-------|
| `shoulderBalance`   | 0.2   |
| `patternMinBars`    | 30    |
| `patternMaxBars`    | 80    |

## マーカー

- 右肩の足下に `arrowUp`、略号 `iH&S`
- 確定時に塗りつぶし、通知発火

## 難度

- 5 ピボット構成・ネックライン検出が必要なため、Double Bottom より実装が重い
- MVP 余力で実装、段階拡張で精度向上
