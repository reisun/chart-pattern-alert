# Ascending Flag（上昇フラッグ）

## 概要

強い上昇のあと、**少し下向き or 横向きの平行チャネル**で揉み合い、再度上抜ける。

```
              /|
             / |  （揉み合い: 平行チャネル）
            /  |←─＼＼＼
           /       ＼＼＼      ＊← ブレイク
          /              ＼＼＼/
         / 推進波（強い上昇）
        /
```

## 意味合い

- 急騰後の利確・休憩
- トレンドが壊れていなければ再上昇しやすい

## 見るポイント

- 旗に入る前の上昇が**強い**（`flagPoleMinPct` 以上）
- 調整が**長すぎず崩れすぎない**
- 上抜けで継続期待

## 検出アルゴリズム

1. 直近で **推進波（pole）** を特定
   - 直近 N 本の中で、連続して上昇している区間の終点→始点差が `flagPoleMinPct` 以上（例 3%）
2. pole 終点から後の本数 M 本を取り、**高値列 / 安値列** にそれぞれ線形回帰
3. 2 本の線の傾きが `flagSlopeMax` 以下、かつほぼ平行（傾き差が閾値以下）→ フラッグ成立
4. チャネルの**上辺を上抜け**で確定（break 足の `close` > 上辺）

## 閾値

| 名称                | 既定   |
|---------------------|--------|
| `flagPoleMinPct`    | 3%     |
| `flagSlopeMax`      | 30°    |
| `flagParallelTol`   | 15°    |
| `flagMinBars`       | 5      |
| `flagMaxBars`       | 25     |

## マーカー

- ブレイク足に `arrowUp`、略号 `A-Flag`
- 旗部分にオーバレイラインを引けるとベター（余力があれば）

## 実装メモ

- pole/flag の区別は連続性判定に工夫が必要
- フラッグを検出できないケースでは Triangle/Wedge に寄せる選択肢あり
